Under the Hood: A Prompt's Journey

Each message goes through 4 steps before a single word is generated.

①

Tokenization

Text → subwords → integer IDs via BPE (GPT) or SentencePiece. This is where the cost is decided: ~1 token ≈ 4 characters

②

Embedding

Each ID → high-dimensional vector (e.g., 4,096d). A positional encoding (RoPE) is added so the model knows word order.

③

Self-Attention

Each token "looks at" all others via Q·K·V matrices (multi-head). O(n²) complexity — doubling tokens = 4× the cost.

④

Generation

Softmax → vocabulary probabilities → 1 token chosen. Added to context, the cycle repeats (autoregressive).

↻ This cycle repeats for each generated token — the entire context is re-read at each iteration

04

Context Challenges

The larger the context grows, the more these phenomena degrade the quality of responses.

🎯 Lost in the Middle

Information placed in the middle of the context is "forgotten". The model favors the beginning and end. (Liu et al., 2023)

🔇 Attention Dilution

More tokens = scattered attention scores. The useful signal is drowned in noise from irrelevant data.

🔄 Context Rot

Back-and-forth prompt↔response exchanges accumulate stale context that pollutes attention in subsequent turns.

📈 Quadratic Cost

O(n²) in memory and computation for attention. 128K tokens costs ~16× more than 32K. Latency and cost explode.

💡

Conclusion: Reducing tokens is not just a cost issue — it's about improving the quality of responses by keeping a clean and focused context.

ℹ️ How are tokens counted?

Our function uses the approximation 1 token ≈ 4 characters. In reality, each provider has its own tokenizer: OpenAI uses tiktoken (BPE), Anthropic has its own tokenizer. The approximation is sufficient to compare relative gains between commands.

Which Tool to Use?

  • Files (names, extensions) → fd
  • Text inside files → rg
  • A field in a YAML → yq
  • A field in an XML → yq -p xml
  • A field in a JSON → jq
  • The same field across N files → fd + yq/jq/xq
  • Files + their content → fd + rg
  • CLI output too verbose → snip
